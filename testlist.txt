Case 1. [엔터 키]를 이용한 메시지 전송
Action (동작):메시지 입력창에 "안녕하세요"를 타이핑한 후, 키보드의 `Enter` 키를 누른다.
Expected Result (기대 결과):
채팅창 리스트에 "안녕하세요"라는 메시지가 추가되어야 함.
메시지 입력창의 값(`value`)은 **완벽히 빈 문자열("")** 상태여야 함. (글자가 남아있으면 안 됨)

Case 2. [전송 버튼]을 이용한 메시지 전송
Action (동작): 메시지 입력창에 "반갑습니다"를 타이핑한 후, 마우스로 '전송' 버튼을 클릭한다.
Expected Result (기대 결과):
채팅창 리스트에 "반갑습니다"라는 메시지가 추가되어야 함.
메시지 입력창의 값(`value`)은 **완벽히 빈 문자열("")** 상태여야 함.

Case 3. [전체 초기화 버튼]
Action (동작): 대화가 진행된 상태에서 '전체 초기화' 버튼을 클릭한다.
Expected Result (기대 결과):
채팅창 내의 모든 사용자 메시지와 챗봇의 이전 응답들이 화면에서 제거되어야 함.
시스템에서 설정한 기본 '웰컴 메시지' 1개만 화면에 남아있어야 함.

Case 4. [응답 전까지 재요청 불가능]
Action (동작): 메시지를 전송하고, 챗봇의 답변이 아직 화면에 나타나지 않은(로딩 중인) 상태에서 다시 한 번 메시지 입력을 시도하거나 전송 버튼을 누른다.
Expected Result (기대 결과):
답변 대기 중에는 '전송' 버튼에 disabled 속성이 활성화되어 클릭할 수 없어야 함.
입력창에 텍스트를 입력하고 Enter 키를 눌러도 새로운 메시지가 채팅창에 추가되지 않아야 함.

Case 5. [로딩 중 표시]
Action (동작): 사용자가 메시지를 전송한 직후, 챗봇이 답변을 준비하는 시간을 관찰한다.
Expected Result (기대 결과):
챗봇의 답변 위치에 '로딩 중'임을 나타내는 아이콘이나 특정 텍스트(예: "답변을 생각 중입니다...")가 나타나야 함.
답변이 도착하는 즉시 해당 로딩 표시가 화면에서 사라지고 실제 답변 내용으로 교체되어야 함.

Case 6. [시간 표시]
Action (동작): 사용자가 메시지를 보내거나 챗봇의 답변을 받았을 때, 말풍선 주변을 확인한다.
Expected Result (기대 결과):
각 메시지(사용자 및 챗봇 모두) 옆에 현재 시각이 HH:mm (예: 21:58) 형식으로 표시되어야 함.
표시된 시간이 실제 메시지 발송 시점과 일치하는지 확인.

Case 7. [스크롤 자동화]
Action (동작): 채팅창 높이를 넘어설 만큼 여러 번 메시지를 주고받아 스크롤바가 생기게 한다.
Expected Result (기대 결과):
새로운 메시지가 추가될 때마다 채팅창의 스크롤이 자동으로 최하단(가장 최근 메시지)으로 이동해야 함.
사용자가 수동으로 스크롤을 올리지 않는 한, 항상 마지막 메시지가 화면에 보여야 함.

Case 8. [쉬프트 줄바꿈]
Action (동작): 입력창에 내용을 적다가 Shift 키를 누른 상태에서 Enter 키를 누른다.
Expected Result (기대 결과):
메시지가 전송되지 않아야 함.
입력 커서가 다음 줄로 넘어가며 입력창 내에서 줄바꿈이 이루어져야 함.

Case 9. [History - 내역 조회 및 입력]
Action (동작): 'History' 버튼을 클릭해 과거 내역 리스트를 열고, 그중 하나의 내역을 클릭한다.
Expected Result (기대 결과):
History 팝업/사이드바가 정상적으로 열리고 과거 대화 리스트가 보여야 함.
특정 내역 클릭 시, 해당 메시지 텍스트가 현재 메시지 입력창(input/textarea)에 자동으로 입력되어야 함.

Case 10. [History - 기록 모두 삭제]
Action (동작): History 메뉴 내의 '기록 모두 삭제' 버튼을 클릭한다.
Expected Result (기대 결과):
History 리스트에 있던 모든 과거 내역 데이터가 화면에서 사라져야 함.
"기록이 없습니다"와 같은 안내 문구가 표시되어야 함.
